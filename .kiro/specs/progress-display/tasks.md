# Implementation Plan

- [x] 1. Foundation: 例外体系と設定値オブジェクトの実装
- [x] 1.1 progress_display の例外体系を定義する
  - 管理対象外タスクへの操作を表す例外、未開始状態での操作を表す例外、`total` 不正値を表す例外を `ValueError` を継承する公開例外として定義する
  - 設定テーブルの値検証失敗のみを表す内部例外を定義する
  - 各例外クラスに日本語の一行docstringを付与する
  - 観測可能な完了状態: 各公開例外をimportして送出・捕捉でき、例外の継承関係（`ValueError`のサブクラスであること）をpytestで確認できる
  - _Requirements: 1.5, 2.4, 3.5, 4.4, 5.2_
  - _Boundary: exceptions_

- [x] 1.2 設定値オブジェクト(ProgressDisplayConfig)を実装する
  - 完了タスクの自動非表示フラグと表示更新頻度を保持するイミュータブルなデータクラスを、両方ともデフォルト値（自動非表示は無効）付きで定義する
  - 観測可能な完了状態: 引数なしでデフォルト値のインスタンスを生成でき、生成後に属性を変更しようとするとエラーになることをpytestで確認できる
  - _Requirements: 4.2, 4.3_
  - _Boundary: types_

- [x] 2. pyproject.tomlからのprogress_display設定読み込み(load_config)を実装する
  - 呼び出し側ディレクトリから親方向に `pyproject.toml` を探索し、`[tool.python_util.progress_display]` テーブルを解析して設定値オブジェクトを構築する
  - テーブルが存在しない場合はデフォルト値の設定オブジェクトを返す
  - TOML解析失敗、または更新頻度に0以下の値が指定されるなどの値検証失敗があった場合は、警告を発してデフォルト値にフォールバックし、例外を外部に伝播させない
  - 観測可能な完了状態: テーブル不在・正常値・型不正値・TOML解析失敗の4パターンで期待通りの設定オブジェクトが返り、異常系では警告が発火することをpytestで確認できる
  - _Requirements: 4.1, 4.4, 4.5_
  - _Boundary: config_loader_

- [x] 3. Core: ProgressDisplayのライフサイクルとタスク管理を実装する
- [x] 3.1 初期化とコンテキストマネージャによる開始/終了を実装する
  - 複数タスクの進捗を単一のコンソール表示領域にレンダリングする表示エンジンを内部に保持し、開始状態を追跡する
  - コンテキストマネージャとして開始・終了できるようにし、ブロックを抜ける際は例外の有無にかかわらず表示を正しく終了させる
  - 初期化時に表示列（スピナー・バー・パーセンテージ・残り時間等）をカスタマイズ可能にする
  - 観測可能な完了状態: `with` 文で進捗表示を開始・終了でき、ブロック内で例外が発生しても表示が正しく終了することをpytestで確認できる
  - _Requirements: 2.1, 2.3, 5.3_
  - _Boundary: ProgressDisplay_

- [x] 3.2 タスク追加(add_task)と入力検証を実装する
  - 説明文・総量・現在値を指定してタスクを追加し、一意なタスク識別子を発行する
  - 総量に0以下の値が指定された場合、および表示が未開始の状態で追加が要求された場合にそれぞれ専用の例外を送出する
  - 追加時点で現在値が総量に到達している場合も後続の完了判定・自動非表示の対象とする
  - 観測可能な完了状態: 複数タスクを追加すると各タスクの説明文・現在値・総量が単一の表示領域に同時に表示され、不正な総量や未開始状態での追加が期待した例外を送出することをpytestで確認できる
  - _Requirements: 1.1, 1.2, 1.4, 2.4, 3.1, 3.5_
  - _Boundary: ProgressDisplay_

- [x] 3.3 タスク更新(update)・完了判定・自動非表示を実装する
  - 現在値の絶対値指定と相対増分指定の両方で進捗を更新できるようにし、更新結果を表示に即座に反映する
  - 管理対象外のタスク識別子への更新、および未開始状態での更新にそれぞれ専用の例外を送出する
  - タスクの現在値が総量に到達した際に完了状態として扱い、自動非表示の設定が有効な場合は表示から取り除く
  - 観測可能な完了状態: 絶対値指定・相対増分指定の双方で進捗表示が即座に反映され、完了タスクが設定に応じて表示から自動的に消えることをpytestで確認できる
  - _Requirements: 1.3, 1.5, 2.4, 3.2, 3.3, 3.6_
  - _Boundary: ProgressDisplay_

- [x] 3.4 タスクの明示的削除(remove_task)を実装する
  - 管理対象のタスク識別子を指定して表示対象から取り除く
  - 管理対象外のタスク識別子が指定された場合に専用の例外を送出する
  - 観測可能な完了状態: `remove_task` 呼び出し後に対象タスクが表示に含まれなくなり、未知の識別子指定時に例外が送出されることをpytestで確認できる
  - _Requirements: 3.4_
  - _Boundary: ProgressDisplay_

- [x] 3.5 イテラブル向け進捗自動更新ヘルパー(track)を実装する
  - for文でイテレートしながら進捗を自動的に更新するヘルパーを提供する
  - 未開始状態での呼び出しに専用の例外を送出する
  - 観測可能な完了状態: ヘルパーでイテラブルを消費すると要素を処理するたびに進捗が自動的に加算され、表示に反映されることをpytestで確認できる
  - _Requirements: 2.2, 2.4_
  - _Boundary: ProgressDisplay_

- [x] 3.6 公開メソッドの型検証を実装する
  - タスク識別子・総量・説明文などの引数に不正な型が渡された場合に型エラーを送出する検証を各公開メソッドに追加する
  - 観測可能な完了状態: 不正な型を渡した各公開メソッド呼び出しで型エラーが送出されることをpytestで確認できる
